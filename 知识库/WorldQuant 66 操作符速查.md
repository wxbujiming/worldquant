## 一、算术类 Arithmetic（15 个）

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **add(x, y, filter=false)** | **加法**——把两个数加一起。开 filter=true 的话，空值(NaN)先当 0 再加，不会因为缺数据就加不出来。 |
| 2 | **sqrt(x)** | **开方**——给 x 开根号，结果永远是非负数。想保留正负号的开方用 signed_power(x, 0.5)。 |
| 3 | **log(x)** | **取对数**——算自然对数（以 e 为底）。数据差太多倍的时候，取个对数能把差距压小，比如股价从 10 涨到 100 和从 100 涨到 1000，取完对数涨幅就一样了。 |
| 4 | **subtract(x, y, filter=false)** | **减法**——x 减 y。多个输入就从左往右挨个减。开 filter=true 的话空值先当 0。 |
| 5 | **signed_power(x, y)** | **带正负号的幂**——"调音量但不换歌"：正的还是正、负的还是负，只是幅度变了。y>1 放大强信号（强者更强），0<y<1 压缩极端值（尾巴被剪）。和 power 的区别：power 会让负变正。 |
| 6 | **fraction(x, y)** | **取小数部分**——x 除以 y，只留小数，不要整数。比如 7/3=2.33…，留 0.33…。常用来算"超出部分"。 |
| 7 | **exp(x)** | **指数**——算 e 的 x 次方，是 log 的逆运算。能把小幅变化快速放大。 |
| 8 | **power(x, y)** | **幂运算**——x 的 y 次方。注意：负数开非整数次方会变 NaN，想保符号用 signed_power。 |
| 9 | **multiply(x, y, filter=false)** | **乘法**——两个数乘一起。开 filter=true 空值当 0。 |
| 10 | **min(x, y, ..)** | **取最小**——几个数里挑最小的那个，至少给 2 个。 |
| 11 | **max(x, y, ..)** | **取最大**——几个数里挑最大的那个，至少给 2 个。 |
| 12 | **inverse(x)** | **倒数**——1 除以 x。x=0 就爆了。 |
| 13 | **densify(x)** | **压缩稀疏分组**——分组字段里有些桶是空的，这个操作把空桶丢掉、只留有数据的桶，让后续计算更快。不常用，碰到分组太碎时才上。 |
| 14 | **abs(x)** | **绝对值**——去掉负号，只看大小。 |
| 15 | **divide(x, y)** | **除法**——x 除以 y。y=0 会出 NaN。 |

## 二、逻辑类 Logical（11 个）

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **and(input1, input2)** | **并且**——两个都为真才返回 1，否则 0。就像"既要吃得好又要花钱少"。 |
| 2 | **equal(input1, input2)** | **相等判断**——俩数一样返回 1，不一样返回 0。 |
| 3 | **or(input1, input2)** | **或者**——任意一个为真就返回 1，都不真才 0。就像"吃得好或者花钱少，满足一个就行"。 |
| 4 | **not_equal(input1, input2)** | **不等判断**——俩数不一样返回 1，一样返回 0。 |
| 5 | **not(x)** | **取反**——真变假、假变真。1 变 0，0 变 1。 |
| 6 | **greater(input1, input2)** | **大于判断**——input1 比 input2 大返回 1，否则 0。 |
| 7 | **negate(x)** | **取负**——正变负、负变正，就是加个负号。跟 not 不同：not 是逻辑翻转(1↔0)，negate 是数学翻转(3↔-3)。 |
| 8 | **is_finite(input)** | **是不是有限数**——不是 NaN、不是无穷大就返回 1，否则 0。用来过滤"坏数据"。 |
| 9 | **is_nan(input)** | **是不是空值**——输入是 NaN 返回 1，否则 0。用来标记缺数据的地方。 |
| 10 | **if_else(input1, input2, input3)** | **如果…就…否则…**——input1 为真时返回 input2，为假时返回 input3。写策略最常用的分支判断。 |
| 11 | **less(input1, input2)** | **小于判断**——input1 比 input2 小返回 1，否则 0。 |

## 三、时间序列类 Time Series（24 个）

> 这类操作符都是"回头看"：在过去 d 天里做统计，每天滚动计算。

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **ts_sum(x, d)** | **过去 d 天求和**——把过去 d 天的值全加起来。比如过去 5 天成交量之和。 |
| 2 | **ts_zscore(x, d)** | **过去 d 天的 Z-score**——今天的值跟过去 d 天的均值比，差了几个标准差？>2 说明今天很不寻常，<-2 也不寻常。 |
| 3 | **ts_std_dev(x, d)** | **过去 d 天的标准差**——衡量数据波不波动，标准差越大越不安分。 |
| 4 | **ts_mean(x, d)** | **过去 d 天的平均**——最朴素地算个移动平均线。 |
| 5 | **ts_scale(x, d, constant=0)** | **时间序列归一化**——在过去 d 天的范围内，把最小值压到 0、最大值拉到 1，其他值按比例落在中间。就像把考试成绩换算成 0-100 分的排名分。 |
| 6 | **ts_rank(x, d, constant=0)** | **时间序列排名**——今天的值在过去 d 天里排第几？返回 0~1 之间的排名分数，0.9 就是"比 90% 的历史日子都高"。 |
| 7 | **ts_quantile(x, d, driver="gaussian")** | **时间序列分位数**——先算 ts_rank，再把排名映射到某种分布上。gaussian 会让极端值更突出，uniform 保持均匀，cauchy 尾巴拉得更狠。比 ts_rank 多一步"分布变换"。 |
| 8 | **ts_returns(x, d, mode=1)** | **收益率**——"最近 d 天涨/跌了多少比例"。mode=1 是简单收益率 (今-d天前)/d天前，mode=2 是对称收益率，分母用俩值的平均，更稳、不怕分母接近 0。做动量策略最常用。 |
| 9 | **ts_regression(y, x, d, lag=0, rettype=0)** | **时间序列回归**——过去 d 天里 y 和 x 的线性关系是什么？rettype 控制返回什么：残差(0)、斜率β(2)、R²(6) 等。残差大 = 定价可能有偏差，β大 = y 对 x 很敏感。 |
| 10 | **kth_element(x, d, k, ignore="NaN")** | **取第 k 个值**——在过去 d 天里，按时间顺序取第 k 个值。常用于回填缺失数据。 |
| 11 | **ts_corr(x, y, d)** | **过去 d 天相关系数**——x 和 y 最近走不走得像？接近 1 就是同涨同跌，接近 -1 就是反向，接近 0 就是各走各的。 |
| 12 | **ts_count_nans(x, d)** | **过去 d 天有多少空值**——数一数最近 d 天缺了几天数据。缺太多的话，基于这段数据的统计就不靠谱。 |
| 13 | **ts_covariance(y, x, d)** | **过去 d 天协方差**——x 和 y 最近一起变的程度。跟 corr 类似，但没做标准化，值的大小受量纲影响。 |
| 14 | **ts_decay_linear(x, d, dense=false)** | **线性衰减加权平均**——"越近的越重要"：最近一天权重最大，越远权重线性递减。比简单平均更能突出近期趋势，还能降低换手率。 |
| 15 | **ts_product(x, d)** | **过去 d 天连乘**——所有值乘一起。常算复合收益率：每天收益率连乘 = 总收益率。 |
| 16 | **ts_delay(x, d)** | **取 d 天前的值**——"时光机"：看看 d 天前这个数是多少。常配合 ts_delta 用。 |
| 17 | **ts_backfill(x, lookback=d, k=1)** | **向前回填空值**——遇到 NaN 就用最近一个有效值顶上。比如周三数据缺了，拿周二的补。 |
| 18 | **ts_av_diff(x, d)** | **当前值 vs 过去均值之差**——今天的值减去过去 d 天的平均值。正数 = 今天比平时高，负数 = 比平时低。最简单的"偏离均值"信号。 |
| 19 | **ts_arg_min(x, d)** | **最小值出现在几天前**——过去 d 天里最低点是哪天？返回距今天数。返回 0 = 今天就是最低，返回 3 = 3 天前是最低。 |
| 20 | **ts_arg_max(x, d)** | **最大值出现在几天前**——同上，不过找的是最高点。 |
| 21 | **last_diff_value(x, d)** | **最近一次不同的值**——过去 d 天里，最近一个跟今天不一样的值。比如连续 5 天都是 10，它能跳过这些找到上一个非 10 的值。适合检测"最近一次变化前的水平"。 |
| 22 | **ts_step(1)** | **天数计数器**——从第 0 天开始每天加 1，纯递增序列。偶尔用来构造时间相关的辅助变量。 |
| 23 | **ts_delta(x, d)** | **变化量**——今天的值减去 d 天前的值。跟 ts_returns 的区别：delta 算的是绝对差（涨了多少），returns 算的是百分比（涨了百分之几）。 |
| 24 | **days_from_last_change(x)** | **距离上次变化过了几天**——这个值上一次"变"是几天前？数字大说明很久没更新了，数据可能不新鲜。 |

## 四、横截面类 Cross Sectional（6 个）

> 这类操作符是"横向比"：同一天，所有股票之间做比较。

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **winsorize(x, std=4)** | **缩尾/掐头去尾**——超过均值±4倍标准差的值，全部拉回到边界上。就像考试把 0 分和 100 分以外的极端成绩都压到 0 和 100，不让个别人拉歪平均。做策略几乎必用，防止一两只妖股搞坏信号。 |
| 2 | **normalize(x, useStd=false, limit=0.0)** | **标准化**——减去全市场均值（中心化），可选再除以标准差（归一化），还可选把结果卡在 [-limit, +limit] 范围内。比 rank 更精细的标准化方式。 |
| 3 | **quantile(x, driver="gaussian", sigma=1.0)** | **横截面分位数变换**——先给所有股票排名，再把排名映射到指定分布上。gaussian 会让两端更极端，cauchy 尾巴更重，uniform 均匀。跟 ts_quantile 的区别：这个是同一天横着比所有股票，ts_quantile 是一只股票竖着比自己的历史。 |
| 4 | **rank(x, rate=2)** | **横截面排名**——同一天所有股票排个名，结果映射到 0~1。最常用的标准化手段，简单粗暴地把"谁大谁小"变成 0~1 之间的分数，不怕极端值。 |
| 5 | **scale(x, scale=1, longscale=1, shortscale=1)** | **缩放仓位**——让所有股票的绝对值之和 = 指定的总仓位规模(scale)。可以单独调多头(longscale)和空头(shortscale)的权重。管"分配多少钱"的。 |
| 6 | **zscore(x)** | **横截面 Z-score**——减去市场均值、除以市场标准差。结果表示"这个值比市场平均水平高/低几个标准差"。跟 rank 类似但保留了数值间距信息。 |

## 五、向量类 Vector（2 个）

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **vec_sum(x)** | **向量求和**——一个字段里包含多个数值（向量），把它们全加起来变成一个数。比如某只股票有 10 个分析师评级，加在一起。 |
| 2 | **vec_avg(x)** | **向量取平均**——一个字段里包含多个数值，取平均变成一个数。比 vec_sum 更常用，10 个分析师评级取个平均分。 |

## 六、转换类 Transformational（2 个）

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **bucket(rank(x), range="0,1,0.1", ..)** | **自定义分桶**——按你指定的区间把数据分成几组。比如 range="0,1,0.1" 就是把排名 0~1 切成 10 个桶（0~0.1, 0.1~0.2, …），也可以用 buckets="2,5,6,7,10" 手动指定边界。分好桶后可以配合 group 操作符做分组中性化等。 |
| 2 | **trade_when(x, y, z)** | **条件交易**——"有信号才动，没信号不动"。y 条件满足时才把 Alpha 更新为 x，否则保持上一次的值。z 条件满足时赋 NaN（相当于平仓）。核心用途：降低换手率——不天天调仓，只在触发条件时才交易。 |

## 七、分组类 Group（6 个）

> 这类操作符都是"组内比"：同一天，只跟同组（同行、同板块等）的股票比。

| # | 操作符 | 大白话 |
|---|--------|--------|
| 1 | **group_scale(x, group)** | **组内归一化**——在每个组内部把数值缩放到 0~1。比如把科技行业内所有股票的 ROE 归一化，方便跨行业比较。 |
| 2 | **group_neutralize(x, group)** | **组内中性化**——"把同行拉到同一起跑线"：每个股票的值减去同组的平均值。结果 > 0 表示比同行强，< 0 表示比同行弱。最常用的去行业偏差手段，避免选出来的 Alpha 只是因为某个行业整体好。 |
| 3 | **group_zscore(x, group)** | **组内 Z-score**——组内中性化的升级版：不光减均值，还除以组内标准差。能看出"比同行强多少个标准差"，组内更精细的比较。 |
| 4 | **group_backfill(x, group, d, std=4.0)** | **组内回填**——"数据缺了问同行的"：用同组其他股票过去 d 天的 winsorized 均值来填补 NaN。比 ts_backfill 更聪明——不只看自己历史，还参考同行。std 参数控制裁剪极端值的力度。 |
| 5 | **group_mean(x, weight, group)** | **组内调和均值**——按指定权重计算组内调和平均。调和平均更偏重小值，不容易被一个大值带偏。 |
| 6 | **group_rank(x, group)** | **组内排名**——只在同组里排，返回 0~1。比如只看科技行业内的 ROE 排名，而不是全市场排名。配合 group_neutralize 使用效果更好。 |

## 🧩 常见搭配速查

| 目标 | 推荐搭配 | 大白话 |
|------|----------|--------|
| 去极端值 | winsorize(x, 4) | 先把妖怪收拾了 |
| 去行业偏差 | group_neutralize(rank(x), industry) | 拉平行业起跑线 |
| 降低换手率 | trade_when(alpha, condition, exit) | 没信号就躺平 |
| 平滑信号 | ts_decay_linear(x, 10) | 近期权重更高 |
| 看相对位置 | rank(x) 或 zscore(x) | 谁大谁小一目了然 |
| 压缩极端值 | signed_power(x, 0.5) | 大值削一点、方向不变 |
| 放大强信号 | signed_power(x, 3) | 强者更强、方向不变 |
| 判断动量 | ts_returns(close, 20) | 最近涨了还是跌了 |
| 检测异常 | ts_zscore(x, 20) | 今天正不正常 |
| 缺数据补值 | ts_backfill(x, 5) 或 group_backfill(x, industry, 10) | 自己补 / 问同行补 |
